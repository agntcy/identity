Deploy PyramID Agent with LangGraph

This tutorial will walk you through deploying the PyramID Agent with LangGraph, based on the reference implementation of LangGraph's React Agent.

Prerequisites

For this tutorial, we will be needing access to the following platforms and APIs:

  • An Okta instance
  • A weatherapi.com API Key
  • A Docker installation

Deploy PyramID Agent

Step 1: Clone the Repository​

Clone the repository from the following link:

git clone https://github.com/cisco-eti/pyramid

Step 2: Setup the Identity Provider (Okta)​

Admins OIDC Application​

  1. Sign-in in Okta as a super admin.
  2. Create an OIDC Single-Page Application for the Admins.
  3. Add the following redirect URIs:
    • http://localhost:3002
  4. Give acccess to your admin users or groups.
  5. Get the {OKTA_ADMINS_ISSUER}, {OKTA_ADMINS_CLIENT_ID} from the OIDC Application.

Managed Directory API Services Application​

  1. Sign-in in Okta as a super admin.
  2. Create an API Services Application for the Managed Directory.
  3. Choose Public Key / Private Key client authentication and generate a new key ppyramid.
  4. Get the PEM Private Key without any spacing or delimeters to get the {OKTA_API_SECRET_KEY}.
  5. Grant the following API Scopes: okta.apps.read, okta.users.read, okta.groups.read.
  6. Grant Super Admin permission to the API Services Application.
  7. Get the {OKTA_API_CLIENT_ID} from the API Services Application.

Example of environment variables​

Example of environment variables

The environment variables should look like this:

  • {OKTA_ISSUER}: https://trial-123456.okta.com/oauth2/default
  • {OKTA_ADMINS_ISSUER}: https://trial-123456.okta.com/oauth2/default
  • {OKTA_ADMINS_CLIENT_ID}: 0oab1cdefghijklmnopq2
  • {OKTA_ADMINS_SUBS}: some_user@your_org.com
  • {OKTA_ORG_URL}: https://trial-123456.okta.com
  • {OKTA_API_CLIENT_ID}: 0oab1cdefghijklmnopq2
  • {OKTA_API_SECRET_KEY}: MIIEvQIBADANBgkq.......

Step 3: Setup the Backend and the UI​

Backend​

This is the minimal configuration required to run the PyramID backend. You can add more configuration options as needed.

########################
# DEFAULT PyramID
########################
DEFAULT_PyramID_ISSUER=`{OKTA_ISSUER}`

########################
# IAM
########################
IAM_ADMINS_ISSUER=`{OKTA_ADMINS_ISSUER}`
IAM_ADMINS_CID=`{OKTA_ADMINS_CLIENT_ID}`
IAM_ADMINS_SUBS=`{OKTA_ADMINS_SUBS}`
IAM_MANAGED_DIRECTORY_ORG_URL=`{OKTA_ORG_URL}`
IAM_MANAGED_DIRECTORY_API_CLIENT_ID=`{OKTA_API_CLIENT_ID}`
IAM_MANAGED_DIRECTORY_API_SECRET=`{OKTA_API_SECRET_KEY}`

UI​

This is the minimal configuration required to run the PyramID UI. You can add more configuration options as needed.

VITE_CCI_UI=`{OKTA_ORG_URL}`
VITE_CCI_OIDC_CLIENT_ID=`{OKTA_ADMINS_CLIENT_ID}`
VITE_CCI_OIDC_ISSUER=`{OKTA_ADMINS_ISSUER}`

Step 4: Deploy PyramID UI, Backend, Docs and DB with Docker​

Use the following command to deploy the PyramID UI, Backend and Documentation with Docker:

  • With MongoDB
  • Without MongoDB
make start_pyramid
make start_pyramid_no_db

Create an Okta Service Account​

  1. Sign-in in Okta as a super admin.
  2. Create an Application OIDC Web Credentials for the Service Account.
  3. Get the {PyramID_SA_CLIENT_ID}, {PyramID_SA_CLIENT_SECRET}, {PyramID_SA_CLIENT_TOKEN_ENDPOINT} from the Service Account.

Step 6: Deploy PyramID Agent​

Configure LLM access​

Setup your OpenAI / AzureAI API keys in the .env file in the agent/src directory:

PyramID_USE_SSL=0

# ---- OpenAI API key ----
OPENAI_API_KEY=

# ---- Azure OpenAI API key ----
OPENAI_API_VERSION=
AZURE_OPENAI_ENDPOINT=
AZURE_OPENAI_API_KEY=

PyramID_CATALOG_ID=`{PyramID_CATALOG_ID}`
PyramID_SA_CLIENT_ID=`{PyramID_SA_CLIENT_ID}`
PyramID_SA_CLIENT_SECRET=`{PyramID_SA_CLIENT_SECRET}`
PyramID_SA_CLIENT_TOKEN_ENDPOINT=`{PyramID_SA_CLIENT_TOKEN_ENDPOINT}`

Deploy with LangGraph​

  1. Install the LangGraph CLI:
pip install --upgrade "langgraph-cli[inmem]"
  1. Install dependencies:
cd ./agent/src
pip install -e .
  1. Launch the LangGraph Server:
langgraph dev
LangGraph Server

API: http://localhost:2024

Docs: http://localhost:2024/docs

LangGraph Studio Web UI: https://smith.langchain.com/studio/?baseUrl=http://127.0.0.1:2024
