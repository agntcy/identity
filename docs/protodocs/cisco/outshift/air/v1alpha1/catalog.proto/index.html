OAuthFlow represents a single OAuth 2.0 flow configuration. It defines the necessary endpoints and scopes required for a specific type of OAuth 2.0 flow.

OAuthFlows​

OAuthFlows represents a collection of supported OAuth 2.0 flows. Each flow provides a different method for acquiring an access token.

NameTypeDescription
implicitoptional OAuthFlowImplicit flow is where the access token is returned directly in the redirect URI fragment, suitable for client-side applications.
passwordoptional OAuthFlowPassword flow allows resource owner credentials (username and password) to be exchanged directly for an access token.
client_credentialsoptional OAuthFlowClientCredentials flow is used by applications to obtain an access token using their client credentials, without user interaction.
authorization_codeoptional OAuthFlowAuthorizationCode flow involves obtaining an authorization code which is exchanged for an access token, suitable for server-side applications.
